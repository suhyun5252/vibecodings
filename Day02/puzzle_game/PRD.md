# PRD: 단계형 카드 매칭 퍼즐 게임

## 1. 프로젝트 개요

브라우저에서 실행되는 카드 매칭 퍼즐 게임을 만든다. 사용자는 시작 화면에서 게임을 시작하고, 쉬운 단계부터 어려운 단계까지 순서대로 퍼즐을 클리어한다. 각 단계는 제한 시간 안에 같은 그림 카드 2장을 모두 맞추는 방식으로 진행된다.

## 2. 목표

- HTML, CSS, JavaScript만으로 실행 가능한 단일 페이지 게임 제공
- 코딩 초보자가 구조를 이해할 수 있는 명확한 코드 구성
- 단계별 난이도 상승을 통해 게임 진행감 제공
- 카드 뒤집기, 점수, 타이머, 게임오버, 종료 화면을 포함한 완성형 게임 경험 제공

## 3. 대상 사용자

- 웹 기초 학습자
- 간단한 브라우저 게임을 만들어보고 싶은 사용자
- HTML/CSS/JavaScript 이벤트 처리와 DOM 조작을 학습하는 초보자

## 4. 핵심 사용자 흐름

1. 사용자는 시작 화면에서 `게임 시작` 버튼을 누른다.
2. 게임은 1단계 쉬움 난이도부터 시작된다.
3. 사용자는 카드를 클릭해 같은 그림 2장을 찾는다.
4. 같은 그림을 맞추면 점수가 10점 증가한다.
5. 현재 단계의 모든 카드 쌍을 맞추면 다음 단계로 자동 이동한다.
6. 모든 단계를 통과하면 종료 화면에서 축하 메시지와 최종 점수를 확인한다.
7. 사용자는 `다시 도전` 또는 `다시 시작`을 통해 1단계부터 새로 시작할 수 있다.

## 5. 게임 단계

| 단계 | 난이도 | 보드 크기 | 제한 시간 |
|---|---|---:|---:|
| 1단계 | 쉬움 | 3x3 | 30초 |
| 2단계 | 중간1 | 4x4 | 60초 |
| 3단계 | 중간2 | 5x5 | 90초 |
| 4단계 | 어려움 | 6x6 | 120초 |

## 6. 기능 요구사항

### 6.1 시작 화면

- 게임 최초 진입 시 바로 게임을 시작하지 않는다.
- `게임 시작` 버튼을 제공한다.
- 버튼 클릭 시 1단계부터 게임 화면으로 전환한다.
- 시작 전에는 타이머가 동작하지 않아야 한다.

### 6.2 게임 화면

- 현재 단계, 난이도, 시도 횟수, 점수, 남은 시간을 표시한다.
- 보드는 현재 단계의 크기에 맞게 자동 생성한다.
- 카드는 클릭 시 3D 회전 애니메이션으로 뒤집힌다.
- 같은 그림 2장을 선택하면 매칭 상태로 고정한다.
- 다른 그림 2장을 선택하면 잠시 보여준 뒤 다시 뒤집는다.

### 6.3 점수 시스템

- 같은 그림 카드 쌍을 맞출 때마다 10점을 추가한다.
- 점수는 다음 단계로 이동해도 유지된다.
- `다시 시작` 또는 `다시 도전` 시 점수는 0점으로 초기화된다.
- 종료 화면에는 최종 점수를 표시한다.

### 6.4 단계 진행

- 게임은 항상 쉬움 난이도부터 시작한다.
- 사용자가 난이도를 직접 선택하지 않는다.
- 현재 단계의 모든 카드 쌍을 맞추면 다음 단계로 자동 이동한다.
- 마지막 단계까지 완료하면 종료 화면으로 이동한다.

### 6.5 제한 시간과 게임오버

- 각 단계는 제한 시간을 가진다.
- 남은 시간이 0초가 되면 게임오버 상태가 된다.
- 게임오버 시 모든 카드는 클릭할 수 없어야 한다.
- 게임오버 메시지와 현재 점수를 표시한다.

### 6.6 빈칸 처리

- 3x3, 5x5처럼 칸 수가 홀수인 단계는 카드 쌍으로 모든 칸을 채울 수 없다.
- 이 경우 가운데 칸을 빈칸으로 둔다.
- 빈칸은 클릭할 수 없고 매칭 대상이 아니다.

### 6.7 종료 화면

- 모든 단계를 통과하면 축하 메시지를 보여준다.
- 최종 점수를 표시한다.
- `다시 도전` 버튼을 제공한다.
- `다시 도전` 클릭 시 1단계부터 새 게임을 시작한다.

## 7. UI 요구사항

- 전체 게임은 중앙 정렬된 카드형 레이아웃으로 표시한다.
- 시작 화면, 게임 화면, 종료 화면은 명확히 구분한다.
- 카드 뒤집기는 3D 회전처럼 보이도록 `rotateY` 기반 애니메이션을 사용한다.
- 모바일 화면에서도 보드와 텍스트가 깨지지 않아야 한다.
- 빈칸은 일반 카드와 구분되도록 비활성 스타일을 적용한다.

## 8. 기술 요구사항

- `index.html` 하나의 파일 안에 HTML, CSS, JavaScript를 작성한다.
- 외부 라이브러리 없이 순수 JavaScript로 구현한다.
- DOM 조작으로 보드, 카드, 화면 전환, 점수, 타이머를 제어한다.
- Fisher-Yates 셔플 방식으로 카드 순서를 무작위화한다.

## 9. 성공 기준

- 시작 버튼을 누르기 전에는 게임과 타이머가 시작되지 않는다.
- 모든 단계가 쉬움, 중간1, 중간2, 어려움 순서로 진행된다.
- 같은 그림을 맞출 때마다 점수가 10점씩 증가한다.
- 제한 시간 초과 시 게임오버가 정상 동작한다.
- 모든 단계를 클리어하면 종료 화면과 최종 점수가 표시된다.
